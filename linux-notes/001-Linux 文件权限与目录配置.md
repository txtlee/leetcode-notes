# 第 5 章 · Linux 文件权限与目录配置
**《鸟哥的linux私房菜》第五章 Linux 文件权限与目录配置**


## 一、使用者与群组(权限的三大身份)

Linux 文件权限以「身份」为单位划分,共三类:
| 身份 | 缩写 | 含义 |
|------|------|------|
| **owner(拥有者)** | `u` | 文件的创建者/所有者 |
| **group(群组)** | `g` | 文件所属群组内的成员 |
| **others(其他人)** | `o` | 既非拥有者也不在群组里的人 |

- **root(超级用户)** 不受权限限制,可读写任意文件。
- 每个账号可属于「多个群组」;但每个文件只有一个拥有者和一个所属群组。
- 相关账号文件:`/etc/passwd`(账号)、`/etc/shadow`(密码)、`/etc/group`(群组)。


## 二、Linux 文件权限概念

### 1. `ls -al` 输出解读

```
-rwxr-xr--  1  root  root  1234  Jul 1 10:00  filename
└──属性──┘ ┊  ┊     ┊     ┊     └──修改时间──┘ └─文件名
        连数 拥有者 群组  大小
```

第一列共 10 个字符:

| 字符位 | 含义 |
|--------|------|
| 第 1 位 | **文件类型**:`-`普通文件、`d`目录、`l`链接、`b`块设备、`c`字符设备 |
| 第 2-4 位 | owner 的权限 `rwx` |
| 第 5-7 位 | group 的权限 `rwx` |
| 第 8-10 位 | others 的权限 `rwx` |

### 2. rwx 对「文件」与「目录」含义不同(重点!)

| 权限 | 对**文件** | 对**目录** |
|------|-----------|-----------|
| `r` | 可读取文件内容 | 可列出目录下的文件名(`ls`) |
| `w` | 可修改文件内容(不含删除) | 可改动目录结构:新建/删除/重命名/移动文件 |
| `x` | 可执行(程序/脚本) | 可进入该目录(`cd`),并能访问其中的文件 |

> ⚠️ 易错点:**目录的 `w` 决定能否删除里面的文件**,而与文件本身的权限无关。即使你对某文件只有 `r`,只要对它所在目录有 `w`,你照样能删除它。

### 3. 权限与数字

`r=4`,`w=2`,`x=1`,相加得权限数字。如 `rwxr-xr--` = `7 5 4`。

## 三、改变权限与属性的核心命令

### 1. `chgrp` —— 改变所属群组

```bash
chgrp groupname filename        # 单个。将文件"filename"的所属群组改为群组“groupname”
chgrp -R groupname dirname      # -R 递归(含子目录)
```

### 2. `chown` —— 改变拥有者(可同时改群组)

```bash
chown user filename             # 改拥有者
chown user:group filename       # 同时改拥有者:群组(冒号或点皆可)
chown .group filename           # 只改群组
chown -R user:group dirname     # 递归
```

> 常用场景:复制文件给他人前需 `chown` 改拥有者,否则对方无权使用。

### 3. `chmod` —— 改变权限(两种写法)

**(1) 数字法**(最常用)

```bash
chmod 755  filename   # rwxr-xr-x
chmod 644  filename   # rw-r--r--
chmod -R 755 dirname  # 递归
```

**(2) 符号法**(精细调整)

```
身份: u / g / o / a
操作: +(加) -(减) =(设定)
权限: r / w / x
```

```bash
chmod u+x  filename       # owner 加执行权
chmod g-w  filename       # group 去掉写权
chmod a=r  filename       # 所有人只读
chmod u=rwx,go=rx filename
```

> 记忆:身份在前,操作符在中,权限在后,如 `u+x`、`o-w`。

### 4. 常见预设权限

- 文件默认 `-rw-rw-rw-`(666) → 经 umask 减后通常为 `644`。
- 目录默认 `drwxrwxrwx`(777) → 经 umask 减后通常为 `755`。
- `umask` 查看默认遮罩:`umask` 显示数字(如 `022`),`umask -S` 显示符号形式。

## 四、目录与路径

### 1. 特殊目录

| 符号 | 含义 |
|------|------|
| `.` | 当前目录 |
| `..` | 上一层目录 |
| `~` | 当前用户的家目录 |
| `~account` | 某账号的家目录 |
| `-` | 上一个工作目录(`cd -`) |

### 2. 路径:绝对 vs 相对

- **绝对路径**:从根 `/` 写起,如 `/home/user/file`。
- **相对路径**:相对当前目录,如 `../dir/file`。

### 3. 路径相关命令

| 命令 | 作用 |
|------|------|
| `cd` | 切换目录 |
| `pwd` | 显示当前绝对路径(`-P` 显示真实物理路径,非链接) |
| `mkdir` | 新建目录(`-p` 递归创建父目录,`-m` 设权限) |
| `rmdir` | 删除**空**目录(`-p` 连同空父目录一起删) |
| `rm -r` | 删除非空目录(谨慎!) |

### 4. `$PATH` —— 执行文件搜索路径

- 系统按 `PATH` 变量中列出的目录顺序查找可执行命令。
- 查看命令实际位置:`which commandname`。
- 添加自定义目录:`PATH="${PATH}:/your/dir"`。
- 不同身份 PATH 不同(root 的 PATH 含 `/sbin`、`/usr/sbin`)。

## 五、文件种类与扩展名

### 1. Linux 常见文件类型(由 `ls` 第一位字符体现)

| 字符 | 类型 | 说明 |
|------|------|------|
| `-` | 普通文件 | 文本、二进制、数据格式 |
| `d` | 目录 | directory |
| `l` | 符号链接 | link,指向另一文件 |
| `b` | 块设备 | block,如硬盘 `/dev/sda` |
| `c` | 字符设备 | character,如键盘、终端 |
| `s` | 套接字 | socket,进程间通信 |
| `p` | 管道 | pipe,FIFO |

### 2. 扩展名(仅供参考,Linux 不靠扩展名判断类型)

- `.sh` 脚本、`.tar/.gz` 压缩包、`.php/.py` 程序等只是惯例。
- 真正决定能否执行的是 `x` 权限,而非扩展名。

## 六、Linux 目录配置标准 —— FHS(Filesystem Hierarchy Standard)

FHS 规定各目录「应存放什么内容」,分三层:可分享/不可分享、可变/不可变。

### 核心目录速查表

| 目录 | 存放内容 | 备注 |
|------|----------|------|
| `/` | 根目录 | 所有目录的起点,与开机/还原/系统修复相关文件放此 |
| `/bin` | 单用户模式下也能用的基本命令 | 如 `cat`、`chmod`、`ls`、`cp` |
| `/boot` | 开机所需文件 | 内核、grub 等 |
| `/dev` | 设备文件 | 如 `/dev/null`、`/dev/sda` |
| `/etc` | 系统主要配置文件 | 文本配置为主,如 `passwd`、`fstab` |
| `/home` | 用户家目录 | `~` = `/home/username` |
| `/lib` | 开机及 `/bin`、`/sbin` 所需函数库 | 含内核模块 |
| `/media` | 可移除设备挂载点 | U 盘、光驱 |
| `/mnt` | 临时挂载点 | |
| `/opt` | 第三方软件 | optional |
| `/root` | root 的家目录 | 注意与 `/` 区分 |
| `/sbin` | 开机/修复/系统设置命令 | 需 root 权限 |
| `/srv` | 服务启动所需数据 | 如 web、ftp 数据 |
| `/tmp` | 临时文件 | 所有人均可写,定期清理 |
| `/usr` | 第二层结构,「Unix Software Resource」 | 类似 Windows 的 `Program Files` |
| `/var` | 第二层结构,变动性数据 | 日志、缓存、邮件队列 |

### `/usr` 与 `/var` 子目录(第二层 FHS)

**`/usr`(软件资源,通常只读):**
- `/usr/bin`、`/usr/sbin`:绝大多数用户/系统命令
- `/usr/lib`:函数库
- `/usr/local`:本机自行安装的软件(建议放此)
- `/usr/share`:共享只读数据(文档、图标)
- `/usr/src`:源码

**`/var`(经常变动的数据):**
- `/var/log`:日志(如 `/var/log/messages`)
- `/var/spool`:队列数据(邮件、打印)
- `/var/cache`:缓存
- `/var/lib`:程序运行状态数据
- `/var/lock`、`/var/run`:锁文件、PID

### 目录树(Directory Tree)

- 整个文件系统是一棵以 `/` 为根的**倒置树**。
- 每个目录都能挂载到不同分区/设备上,逻辑上统一为一棵树。
- 查看目录树结构可用 `tree` 命令。

## 七、重点速记清单

1. **三种身份**:owner / group / others,root 不受限。
2. **rwx 含义随文件/目录而变**——目录的 `w` 能删文件、`x` 才能进入。
3. **三命令**:`chgrp`(群组)、`chown`(拥有者)、`chmod`(权限,数字法 + 符号法)。
4. **数字法**:`r=4 w=2 x=1`,常用 `755`/`644`。
5. **默认权限受 `umask` 影响**:文件常 644、目录常 755。
6. **PATH** 决定命令能否直接执行,`which` 查命令位置。
7. **FHS 五大目录**:`/bin`(命令)、`/etc`(配置)、`/home`(用户)、`/usr`(软件)、`/var`(变动数据)。
8. **目录树**:以 `/` 为根,逻辑统一,物理可分散挂载。

---

## 八、常见易错点回顾

- ❌ 以为文件 `r` 权限就能删除它 → 实际删除取决于**目录**的 `w`。
- ❌ 以为 `chmod` 能改拥有者 → 改拥有者要用 `chown`。
- ❌ 以为 `rmdir` 能删非空目录 → 只能删空目录,非空用 `rm -r`。
- ❌ 以为扩展名决定可执行 → 真正看的是 `x` 权限。
- ❌ 以为 `cd` 进目录只需 `r` → 进入目录需 `x`,列出文件名需 `r`,二者缺一不可。
